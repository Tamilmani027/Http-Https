# HTTP vs. HTTPS

This document provides a technical comparison between HTTP and HTTPS, focusing on security implications and infrastructure architecture for software engineers.

## Fundamental Difference

*   **HTTP (Hypertext Transfer Protocol):** A request/response protocol that transmits data in **plain text**. This makes it vulnerable to interception, tampering, and "man-in-the-middle" attacks.
*   **HTTPS (Hypertext Transfer Protocol Secure):** HTTP combined with **SSL/TLS (Secure Socket Layer/Transport Layer Security)**. It introduces a cryptographic layer that encrypts data in transit, ensuring privacy and server authenticity.
